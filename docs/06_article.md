We have created a universal format for the parsed news that exit the saffron and stored to the database, and we named `Article`.
The article itself is in `JSON` format.

## Saffron data

### `id`

This field is assigned by saffron, and it  is random. It is used to identify a specific article.
All article ids starts with the prefix `art_`.


### `hash`

This field is assigned by saffron, and it is based on the article's data. It is used to identify the article's data, so saffron will
be able to check if an article already exists in database. `hash` field is generated by the following code:

```js
hash(`${this.title} ${this.content} ${this.extras?.toString()} ${this.source.id}`)
```

### `timestamp`

This field is assigned by saffron. It contains the date in milliseconds where the article entered the database.

### `source`

This field is assigned by saffron. It contains the source's file unique generated id and name.


## Article data

### `title`

This field is assigned from the website. It contains the new's title.

### `content`

This field is assigned from the website. It contains the new's content.

### `link`

This field is assigned from the website. It contains the new's link where it redirects to the actual post in the website.

### `pubDate`

This field is assigned from the website. It contains the new's publication date.

### `attachments`

This field is assigned from the website and saffron. It contains the new's attachments and the extracted urls that exists inside the `content`.

### `categories`

This field is assigned from the website and saffron. It contains the new's categories and the `alias` field if it is set.

### `extras`

This field is assigned from the website and saffron. It contains all the extra fields that was parsed and not assigned to a specific field.
